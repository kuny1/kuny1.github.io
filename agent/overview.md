# Agent搭建：选择适合自己的学习方式

## 核心原则：快就是慢，慢就是快
- 不急于求成：基础不扎实，上限低，后期愈发吃力，需要返工。
- 不强行吃苦：上来就研究机器学习理论，Agent 架构设计，自信心很容易被浩如烟海的复杂度摧毁
- 精计划丶重实践：基于开发要求制定精益的分阶段学习计划，先建立初始的连接，再由易到难，逐步建立自信，逐步加强

## 精益的计划
### 1. Agent感性认识：了解并建立对 AI Agent 的第一印象
- 了解 Agent、AI Agent 是什么
- 了解 AI Agent 有哪些具体应用的示例（典型场景）

**验收**：记学习笔记（完善的介绍文档已经非常多了，只需要将自己的理解以学习笔记的方式记录下来即可）

### 2. Hello world: 一个真正能跑的 Demo, 看得见摸得着
- 架构确定：ReAct；理由：足够简单、完美覆盖 Brain（大脑）、Perception（感知）、Action（行动）三个核心模块
- 核心功能：响应外部的输入、思考决策、执行（调用工具、输出内容）
- 具体需求：
  - Perception：用户在命令行输入一个自然语言问题，例如“北京天气怎么样？”或“123 + 456等于多少？”
  - Brain：LLM决定调用哪个工具、如何组织答案（get_weather、calculate）
  - Action：执行工具 → 将结果返回LLM → LLM生成最终答案

**验收**：成功运行一个本地的 Agent

### 3. Python 基础：Life is Short, you need python.
- Python是一门语言，最基本的任务可以分为3个：
1. 存储数据 — 在内存中记住某个值
2. 操作数据 — 对存储的值做运算（数学、逻辑、文本处理）
3. 控制流程 — 决定下一步做什么（条件、重复、跳转）

- 相对应的需要掌握的基础是：
1. 存储数据：值与类型的表示 → 数据存储的最小单元
2. 操作数据：操作符 → 对数据进行操作
3. 控制流程：控制流语句 → 决定程序的走向
4. 复合应用
   - 数据结构：组织多个数据
   - 函数：操作数据和控制流程的抽象与复用
   - 输入与输出：与外部产生交互
   - 模块与包：基于函数及输入输出的更复合的组合

**验收**：学习笔记、跟已经掌握的语言的核心差别等
