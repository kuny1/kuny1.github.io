# 初识 AI Agent

## Agent 的广义定义

第一性定义：一个能够自主地感知环境、处理信息并采取行动以达成目标的实体

### 核心要素
1. 自主性：不受外部直接指令控制，能独立运行
2. 感知能力：获得环境状态（传感器、输入数据）
3. 目标导向：存在内在目标或效用函数，驱动行为选择
4. 行动能力：改变环境状态（执行器、输出指令）

## AI Agent（抽象定义）

第一性定义：使用人工智能技术（特别是学习与推理）来实现感知、决策、行动循环的自主实体

### 组成部分
1. 感知：输入是自然语言 / 多模态提示，LLM 将世界状态编码为 token 序列
2. 决策：LLM 作为“中央处理器”，通过对话或链式思维模拟推理过程，生成行动计划（调用工具、编写代码、回复文本）
3. 行动：输出可以是对外部 API 的函数调用、对环境的自然语言指令、或直接执行的动作
4. 长期记忆：增加外部向量数据库或对话历史窗口，模仿人类记忆

### 跟传统 Agent 的区别
1. 感知能力：更泛化，更贴近真实人类社会（隐式学习了世界模型）
2. 决策：AI Agent 决策由机器学习（特别是强化学习或监督学习）从数据或交互经验中自动生成（传统 Agent 决策由人显式编写）
3. 目标：自然语言可变目标，可复合、可干预


## AI Agent（技术本质）
1. Agent 是一个从“感知历史”到“行动”的函数
- **Aₜ = π(s≤ₜ)**
  - 截止当前时刻 t 的全部感知历史（包括内部状态）
  - 决策函数（或称策略）
  - Aₜ：此刻输出的行动

1. 传统 Agent：π 是人工显式构造的
2. AI Agent：π 是从数据中自动学习的（也称：训练、回归）
- 能泛化：因为学习过程从数据中提取了统计规律，用于未见输入
- 可能涌现复杂策略：因为优化过程可能发现人工难以写出的映射
- 不可解释：是分布式向量权重，不是符号规则
1. LLM Agent：π 是 大语言模型本身

## LLM Agent 基本概念

### “快递小哥”的例子
工作的本质：在订单和城市的规则里（环境），通过看路看手机（感知），决定走哪条路、先送哪单（决策），然后骑车按导航行动（行动），最终把餐准时送达（达成目标）

- 环境：城市道路、交通规则、订单系统
- 目标：准时送达，获得好评
- 感知：看路况、看手机导航、听系统派单
- 决策：规划路线、判断先取后送的顺序、遇到堵车要不要绕路
- 行动：骑车、打电话、按门铃、点击“已送达”

| 定义要素 | 外卖小哥               | LLM Agent                                        |
| -------- | ---------------------- | ------------------------------------------------ |
| 环境     | 城市道路、商家、顾客   | 文本界面、工具、互联网、数据库                   |
| 目标     | 准时送达               | 用户指令：“帮我写一份竞品分析报告”               |
| 感知     | 眼睛看路、耳朵听导航   | 读取用户输入、工具返回的结果、检索到的文档       |
| 决策     | 大脑判断路况、选择路线 | LLM 推理：接下来该做什么？先查资料还是先搭框架？ |
| 行动     | 骑车、打电话           | 生成文字、调用搜索工具、执行代码、发送请求       |

### 概念&技术属于哪一部分

| 核心维度 | 概念/技术                         | 它在这个维度里扮演的角色                                                                                                                |
| :------- | :-------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------- |
| **环境** | **System Prompt（系统提示词）**   | Agent 的“基本法”和“角色说明书”。定义了它的身份、能做什么、不能做什么，是所有决策和行动的底层约束。                                      |
|          | **Context Window / 对话历史**     | Agent 的“当下世界”。它只能看到这个窗口里的内容，这是它最直接的工作环境。                                                                |
|          | **向量数据库 / 知识库**           | Agent 的“外部图书馆”。存储着可以被检索到的长期知识，是环境的一部分，可以按需查阅。                                                      |
|          | **可用工具集（Tool List）**       | Agent 的“武器库清单”。告诉 Agent 环境中存在哪些它可以调用的功能（搜索、计算器等）。使用**MCP**协议调用                                  |
|          | **Human-in-the-loop（人机协同）** | 环境中的“特殊干预者”。在关键时刻，人的反馈或批准会直接修改环境状态或决策方向。                                                          |
| **目标** | **用户目标（Agent 对齐后）**      | 决策核心消化用户输入后，结合 System Prompt 与环境约束，形成的、驱动本次任务循环的明确内部目标状态。是决策的产物，而非感知的输入。       |
|          | **任务分解（Plan & Execute）**    | 目标的“细化工厂”。把一个复杂的大目标拆解成一系列有序的小目标（“先搜集资料，再列大纲…”）。                                               |
|          | **子目标生成**                    | 在任务执行中动态派生的新目标。比如 AutoGPT 在任务列表里自己添加的下一步待办事项。                                                       |
| **感知** | **用户输入（User Prompt）**       | 感知的“文字听觉”。接收到的、一切来自用户的、原子性的原始文本信号，是物理输入的第一触点。                                                |
|          | **工具返回结果（Observation）**   | 感知的“触觉反馈”。比如搜索引擎返回的网页摘要、计算器返回的数字。这是 Agent 观察自身行动结果的关键。                                     |
|          | **RAG 检索结果**                  | 感知的“记忆读取”。从外部知识库拉取相关文档片段，作为增强的上下文注入到感知中。                                                          |
|          | **多模态输入**                    | 感知的“其他感官”。让 Agent 能“看见”图片（GPT-4V）、“听见”声音（Whisper），理解更丰富的环境信息。                                        |
| **决策** | **LLM 核心**                      | 决策的“大脑皮层”。所有推理、规划和选择行动的最核心处理器。                                                                              |
|          | **CoT / ToT / GoT（思维链等）**   | 决策的“思考方法论”。指导大脑如何一步步推演（Chain）、多路探索（Tree）、还是网状联想（Graph）。                                          |
|          | **ReAct（推理-行动循环）**        | **决策的核心节拍器**。它把决策过程固化为一个标准循环：在“思考下一步该做什么”和“执行行动并观察结果”之间交替。                            |
|          | **规划器（Planner）**             | 决策的“战略部”。在动手前，先完整地想好整个行动计划，生成一系列步骤。                                                                    |
|          | **Reflexion / Self-Critique**     | 决策的“纠错委员会”。根据行动结果进行自我反思，生成口头化的教训，并反馈到下一次决策中，让 Agent 变得更聪明。                             |
| **行动** | **Function Calling（函数调用）**  | **行动的标准“动作指令”**。Agent 决定行动后，输出的不再是自由文本，而是一个结构化的函数名和参数，用以精准触发外部工具。                  |
|          | **工具使用（Tool Use）**          | 行动的具体“招式”。包括调用搜索引擎、执行代码解释器、操作浏览器、发送 API 请求等一切改变外部世界或获取信息的行为。最典型的代表**Skills** |
|          | **最终文本输出**                  | 行动的“语言表达”。当 Agent 决定任务完成或需要与用户沟通时，生成最终的自然语言回复。                                                     |
|          | **多 Agent 委派**                 | 行动的“任务外包”。一个 Agent 可以将子目标作为指令，发送给另一个专业 Agent 去执行，这本身就是一种特殊行动。                              |

## LLM Agent 常见应用
分类维度
- 环境：封闭、开放
- 计划：提前设计、Agent理解需求并实时生成

① 确定性自动化：封闭 * 提前设计
② 策略性优化：封闭 * 自主规划
③ 引导式探索：开放 * 提前设计
④ 自主智能体：开放 * 自主规划



| 定义要素   | 核心价值                 | 代表产品/能力                                                                                                                                                                   |
| ---------- | ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 自动化脚本 | 可靠、快速、零差错       | UiPath、传统RPA、SQL存储过程                                                                                                                                                    |
| 策略引擎   | 在已知规则中找到最优解   | AlphaGo、量化交易Agent、芯片布线优化                                                                                                                                            |
| 智能副驾   | 增强人的能力，不替代决策 | **AI编程协作**：Claude Code， Cursor， Windsurf； **通用对话/信息处理**：DeepSeek， ChatGPT， 腾讯WorkBuddy； **设计创意生成**：Midjourney， Adobe Firefly                                |
| 智能代理   | 替代人完成开放式任务     | **通用自主任务**：AutoGPT, BabyAGI；**跨系统自主代理**：OpenClaw， OpenAI Operator；**自进化智能体**：Hermes Agent  **持续后台运行**：Tembo（异步编程）； **自主编程任务**：Devin（AI全栈工程师） |
